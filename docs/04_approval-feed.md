# Phase 1-D: 承認システム・フィード表示（Day 7-8）

## 目的
コミュニティ承認システムと、承認待ち投稿を混ぜたフィード表示の実装

## タスク

### 1. 投稿カードコンポーネント（`post-card.tsx`）
Stitchデザインに基づく:
- [x] サムネイル画像（再生ボタンオーバーレイ）
- [x] VTuberアバター + 名前
- [x] 投稿タイトル / おすすめコメント
- [x] 種別バッジ（本配信 / 切り抜き / Xクリップ）
- [x] アクションボタン: ♡お気に入り数 / 👍いいね数 / 🔖ブックマーク
- [x] 投稿者名 + タイムスタンプ
- [x] **承認待ち投稿の場合**:
  - 「🔍 承認待ち」バッジ表示
  - [✅ 承認する] [❌ 不適切] ボタン表示（いいね等の代わりに）
  - 「あとN人の承認で公開されます」プログレス表示

### 2. ホームフィード（`/` ページ）
- [x] 公開済み投稿の一覧表示（新着順デフォルト）
- [x] 承認待ち投稿をフィードに混ぜて表示
  - お気に入りVTuber関連の承認待ちを優先
  - 自分の投稿は除外
  - 既に承認/報告済みの投稿は除外
- [ ] 無限スクロール（Intersection Observer）
- [x] 読み込み中のスケルトンUI

### 3. 承認処理
- [x] 承認ボタン押下 → `approvals` テーブルにINSERT（action: 'approve'）
- [x] 不適切ボタン押下 → `approvals` テーブルにINSERT（action: 'reject'）+ 理由入力（任意）
- [x] DBトリガー `handle_approval()` により:
  - 承認数が必要数に到達 → ステータスを `published` に自動昇格
  - 投稿者の `approved_posts_count` を更新
  - 信頼度レベルの自動昇格チェック（10件→trusted、50件→senior）
  - 不適切報告2件到達 → ステータスを `rejected` に
- [x] 楽観的UI更新（即座にボタン無効化、バックグラウンドでDB更新）

### 4. VTuber詳細ページ（`/vtuber/[id]`）
Stitchデザインに基づく:
- [x] ヒーローセクション（アバター、名前、事務所/グループ）
- [x] 統計: おすすめ数、お気に入り数
- [ ] ♡お気に入りボタン（トグル）
- [ ] タブ: おすすめ一覧 / 人気順 / 新着順
- [x] そのVTuberの承認待ち投稿を上部にまとめて表示
- [x] 関連VTuber（同じ事務所/グループ、横スクロール）

### 5. 信頼度レベル表示
- [ ] マイページに信頼度バッジ表示（新規 / 信頼ユーザー / 上級ユーザー）
- [x] 承認権限の有無に応じたUIの出し分け
  - 新規ユーザー: 承認ボタン非表示
  - 信頼/上級ユーザー: 承認ボタン表示

## 完了条件
- [x] ホームフィードに公開済み + 承認待ち投稿が混在表示される
- [x] 承認/不適切ボタンが正しく動作する
- [x] 必要承認数に到達すると投稿が自動的に公開される
- [x] 信頼度レベルが正しく昇格する
- [x] VTuber詳細ページが表示される
