作成日: 2026-02-26

## エグゼクティブサマリー

### サービス概要
お気に入りのVTuberの過去配信・切り抜き・Xクリップを「おすすめ」として投稿・共有できるWebアプリ。
「VTuber配信の食べログ」として、ファンのUGCによる神回データベースを構築する。
ホスティングは一切せず、リンク共有のみで権利リスクを最小化。

### 市場機会
- VTuber国内市場: 1,260億円（2025年度、前年比120%）
- ファンの68.6%が「推し」を持ち、布教意欲が高い
- **「過去配信おすすめ共有」特化サービスは空白地帯**（ココスキ終了後、有力な後継なし）
- ココスキが1.5ヶ月で13,000投稿・838万PVを達成 → ニーズの強さが実証済み

### 収益目標: 月10〜20万円
- **達成シナリオ**: MAU 10,000〜30,000人（1年以内に到達可能）
- **ハイブリッド収益**: サブスク（主力）+ 広告 + アフィリエイト + 支援
- **損益分岐点**: MAU約3,000人

## 2. コア機能（MVP）

### Phase 1（2週間）: 最小限の価値提供
1. **おすすめ投稿（マルチソース）**: URL入力で自動判別（YouTube本配信/切り抜き/Xクリップ）→ メタデータ自動取得 → コメント+タグ+評価で投稿。切り抜き→元配信の紐づけも可能
2. **コミュニティ承認システム**: 不適切なリンク登録を防ぐ品質管理。新規ユーザーは3人の承認が必要、信頼ユーザー（承認済み10件+）は1人、上級ユーザー（50件+）は即公開。承認待ち投稿はフィードに混ぜて表示し、お気に入りVTuber関連を優先表示
3. **3つのアクション**: ♡お気に入り（VTuber自体）/ 👍いいね（投稿）/ 🔖ブックマーク（投稿）
4. **絞り込み**: 事務所別 / グループ別 / VTuber別 / コンテンツ種別（本配信/切り抜き/Xクリップ）/ タグ別
5. **ソート**: いいね多い順 / 新着順
6. **SNS共有**: OGP画像自動生成（種別バッジ付き）+ Twitter/X・LINE共有（公開済み投稿のみ）
7. **ユーザー認証**: Google/X OAuth

### Phase 2（+2週間）: ソーシャル拡充 + PWA
- お気に入りVTuber新着フィード（承認待ち投稿の優先表示含む）
- 元配信⇔切り抜きの紐づけ表示
- 検索、トレンドランキング、通知
- **PWA化**: ホーム画面追加、オフラインキャッシュ、Web Push通知（Android）

### Phase 3（+1ヶ月）: 成長拡張
- **「このVTuberが好きな人はこの人も好き」レコメンド**（お気に入りデータの協調フィルタリング）
- 布教リスト機能、コメント、マルチプラットフォーム、管理画面

### Phase 4（1年〜）: ストアアプリ化
- **Capacitorで既存コードをラップ** → App Store / Google Play公開（コード書き直し不要）
- ネイティブプッシュ通知（iOS含む）、ディープリンク

## 3. 技術スタック

| レイヤー | 技術 | 理由 |
|---------|------|------|
| フロントエンド | **Next.js 15** (App Router) | Vercel完全統合、OGP生成、RSC |
| BaaS | **Supabase** (PostgreSQL) | RLS、低コスト、ベンダーロックイン低 |
| ホスティング | **Vercel** (Hobby→Pro) | Next.js完全サポート |
| 画像ストレージ | **Cloudflare R2** | 10GB無料、エグレス無料 |
| CDN | **Cloudflare** (Free) | 無制限CDN、DDoS防御 |
| UI | **Tailwind CSS v4 + shadcn/ui** | 高速開発 |
| 外部API | **YouTube Data API v3** | YouTube動画メタデータ取得 |
| 外部API | **X oEmbed API** | Xクリップのメタデータ取得（認証不要） |
| スマホ対応 | **モバイルファーストUI → PWA → Capacitor** | Phase 1でモバイルUI、Phase 2でPWA、Phase 4でストア公開 |

### データモデル概要（11テーブル）
```
agencies（事務所）→ groups（グループ）→ vtubers（VTuber）
profiles（ユーザー: 信頼度レベル付き）→ posts（投稿: 承認ステータス付き）
approvals（✅承認/❌不適切報告）← DBトリガーで自動ステータス昇格
favorites（♡お気に入り: ユーザー⇔VTuber）
likes（👍いいね: ユーザー⇔投稿）
bookmarks（🔖ブックマーク: ユーザー⇔投稿）
tags, post_tags（タグ）
```

## 6. グロース戦略

### バイラル設計の核
1. **推し布教リスト**: おすすめ配信5〜10本をまとめて1URLで共有
2. **魅力的なOGPカード**: サムネ+星評価+推薦コメント+種別バッジ付きカード
3. **布教バッジ/実績**: UGC投稿を動機づけるゲーミフィケーション
4. **ランキング/トレンド**: 週間神回ランキング（いいね順）でリピート促進
5. **レコメンド**: 「このVTuberが好きな人はこの人も好き」で新たな出会い → お気に入り登録 → さらにレコメンド精度向上の好循環

### 獲得チャネル（優先度順）
1. **X(Twitter)**: VTuberファンの主要SNS、OGP最適化、Bot運用
2. **SEO**: 「[VTuber名] おすすめ配信」「神回」系キーワード
3. **Discord Bot**: `/recommend`コマンドで50サーバー以上に浸透
4. **Reddit**: r/Hololive(150万人)等で価値提供ファーストの露出
5. **5ch/まとめサイト**: API/ウィジェット提供でバックリンク獲得

## 8. リスクと対策

| リスク | 影響度 | 対策 |
|--------|--------|------|
| YouTube API規約変更 | 高 | メタデータ中心設計（動画ホストしない）、キャッシュ戦略 |
| 事務所からの差し止め | 高 | ガイドライン厳守、配信リンクのみ、転載しない |
| にじさんじ切り抜き登録制 | 低 | 切り抜きを作成しないため直接影響なし |
| 競合サービス出現 | 中 | UGC蓄積による先行優位、コミュニティがモート |
| ファンコミュニティ炎上 | 中 | 宣伝色を出さない、価値提供ファースト |
| ユーザー離れ | 中 | 継続的な機能更新、季節イベント |
